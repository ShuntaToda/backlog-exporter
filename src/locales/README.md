# Locales

[i18next](https://www.i18next.com/) を使用した多言語対応のための翻訳ファイルを管理するディレクトリです。

## 対応言語

| ファイル  | 言語   |
| --------- | ------ |
| `ja.json` | 日本語 |
| `en.json` | 英語   |

## ネームスペース構造

```
├── commands/
│   ├── all/          # all コマンド
│   ├── issue/        # issue コマンド
│   ├── document/     # document コマンド
│   ├── wiki/         # wiki コマンド
│   └── update/       # update コマンド
│       ├── description   # コマンドの説明文
│       ├── examples/     # ヘルプに表示する使用例
│       ├── labels/       # メッセージに埋め込む名詞ラベル
│       ├── messages/     # 実行時のログ・エラーメッセージ
│       └── args/         # コマンド引数の説明
│
└── common/
    ├── flags/        # 複数コマンドで共有するCLIフラグの説明
    └── messages/     # 複数コマンドで共有するログ・エラーメッセージ
```

## 翻訳の対象範囲

翻訳の対象は **CLIの出力（コンソールメッセージ・ヘルプ・フラグ説明・エラー）だけ** です。以下は意図的にロケールへ依存させず、**日本語固定**にしています。

- **エクスポートされるMarkdownファイルの内容**（`## 基本情報`・`## 詳細`・`## コメント`・`ステータス:` などの見出し・ラベル）
  - ロケールの異なるマシンで `update` を実行するたびにファイル差分（見出しの書き換え）が発生するのを防ぐため
  - 既存のエクスポート済みファイルとの整合性を保つため
- **`backlog-update.log` に書き込むログエントリ**
  - 監査ログとして内容が実行環境に依存しない方が扱いやすいため

エクスポート内容のラベルに `t()` を使わないでください。

## ロケールの決定

環境変数 `LC_ALL` → `LC_MESSAGES` → `LANG` の順で検出します。**いずれも未設定の場合は日本語（`ja`）** です（i18n導入前の既存動作を維持するため。Windowsのcmd/PowerShellは通常 `LANG` を設定しません）。環境変数が設定されていて未対応のロケールだった場合は英語（`en`）にフォールバックします。

## キーの配置基準

### `commands.<command>.*` vs `common.*`

- そのコマンドでしか使わないキーは `commands.<command>.*` に配置する
- 複数のコマンドで横断的に使われるキーは `common.*` に配置する

### サブカテゴリの使い分け

| カテゴリ   | 用途                                         | 例                               |
| ---------- | -------------------------------------------- | -------------------------------- |
| `labels`   | メッセージに埋め込む名詞ラベル                 | `document`, `wiki`                |
| `messages` | 実行時にコンソールに表示するメッセージ         | `fetchStart`, `completed`         |
| `examples` | `--help` に表示するコマンド使用例の説明        | `saveIssues`, `outputDir`         |
| `flags`    | CLIフラグの説明文                              | `apiKey`, `domain`                |

### キーを追加するときの判断フロー

1. そのキーは特定のコマンドでしか使わないか？
   - Yes → `commands.<command>` 配下に追加
   - No → `common` 配下に追加
2. キーの用途は何か？
   - メッセージに埋め込む名詞ラベル → `labels`
   - 実行時メッセージ → `messages`
   - CLIフラグの説明 → `flags`
   - 使用例の説明 → `examples`
